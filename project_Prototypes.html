Delivery Quest System

Demonstration Video


About the project:

In this prototype I wanted to create a system in which the player could collect delivery quests from different points and complete them in any order. Completed quests will get stored in an archive and the player can track a quest to see the destination on a compass.


The system is based on scriptiable objects which hold the quest information, a player quest manager which checks current and archived quests and mailboxes from which the player can take or deliver quests. The UI is all based around checking the player quest manager and displaying current/archived quests accordingly. Below is the player quest manager. Showcasing the entire system here would be too long but I am more than happy to show the project files on request!
