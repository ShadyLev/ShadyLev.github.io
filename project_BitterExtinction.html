Custom Key Binding

This system allows the player to customize the keybindings to their likings. It works on the old unity input system.

What does it include?

Input manager with all key binding elements.

Key object and values.

Controls menu structure.

Overlay panel to choose a key.

The custom key binding system is quite simple in itself. The menu contains a key container where all the key objects are. When player clicks on them a new overlay opens and player can assign a new key to that binding. Player can then save the new configuration or load the previous one. There's also an option to reset the keybindings to the default values.

The Controls Manager Script.

The ControlsAndStatus script holds two dictionaries, one storing default key binding values and second one changable storing the player controls. Additionaly we have the control enums which define a key type that shows what action a key executes. Lastly this script holds a simple function which changes a value in the dictionary to a new one.

The Input Manager Script.

The input manager script handles MOST the logic for keybinding. Here's how it works:

The keys list holds all gameobjects in the menu which show keybindings. And the seleckKeyPanel is the gameobject which shows up when a player click one of the the keybinding gameobjects as you can see in the gif above. Next in the Awake() function we load keybindings from memory in case any were saved in previous sessions.

The OnGUI function listens for any of the key gameobject and waits until the player selects one. When that happens the seleckKeyPanel gets shown with a promp for the player to choose a new keybinding.

Next two functions handle saving and loading of controls.

SaveControlKeybindings simply goes through each keybinding in the ControlsAndStatus player controls dictionary and saves them as an int representation in the playerprefs file.

The LoadControlKeybindings just like the save function goes through the keybinding dictionary comparing each key with the saved list in playerprefs and if a key is saved it means it got changed by the player previously. If that is the case then we change that key to the value stored in playerprefs file. At the end we reset the key icons to update them to their new values.

The ResetKeyBinds function goes simply resets the player keybinding dictionary with the saved default value dictionary.

Object that hold key icons.

The last three functions are very simple. Firstly ResetKeyText uses the ChangeKeyCodeToNormalText function which is a method that simple has "if" statements for some keyboard keys that have weird keycode name (e.g "1" = Alpha1) and changes the text on the key to a normal text. ChangeKeyIcon gets a reference to an object which hold all key icons (see above) and sets the icon that matches the keycode.

Choose a kay panel.

The last part of the system is the panel that actually detects the key input and changes the dictionary.

This object hold the ChooseAKey script which handles the key detection. Firstly the OnGui function checks which key the player wantes to edit then it wait half a second (to remove unwanted inputs from the player) and after that time it waits for a key input.

The WaitForKeyMouse function checks if a keyboard event has occured. When it has it takes the pressed key, changes the keybinding dictionary to the new selected key and closes the overlay. In case of a mouse click we check which mouse button got clicked and change the text to a simpler (M1,M2,M3) and then change the key as normal.

This system is quote simple and not very optimised but it gives player more accesibility and freedom which is very important to reach more audience and improve gameplay.
